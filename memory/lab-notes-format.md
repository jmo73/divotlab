## Beehiiv HTML rules
- Inline CSS only — no <style> blocks
- Table-based layout only — no divs
- Font stack: 'DM Sans', Helvetica, Arial, sans-serif (body) / Georgia, serif (display)
- No <html>, <head>, <body> tags — content only
- Max ~700 lines per HTML block before Beehiiv cuts off
- Split long emails into 3 chunks, each pasted as a separate HTML block

## Voice
- Direct, data-first, intelligent — like a sharp analyst
- No exclamation points ever
- No emojis
- Player last names after first mention
- "Strokes gained" not "stat" — "the field" not "the competition"
- When wrong about a prediction, say so directly — builds trust
- Never invent statistics — use [VERIFY] if data isn't confirmed

## Tuesday Lab Notes structure (free tier)
1. Opener — 2 sentences, most interesting data angle of the week
2. Tournament Preview — 1 paragraph + stats card
3. Players to Watch — 3 players with CF scores and tracking chips
4. Stat of the Week — one big number, explained
5. Last Week Recap — the trust-builder, include almost every week
6. If You Play — one amateur-applicable insight, 1–2 paragraphs
7. Footer CTA — Lab Picks Pro upsell + forward CTA

## Wednesday Lab Picks structure (Pro tier)
1. Pro tier banner + This Week's Approach (2–3 sentences)
2. 4–5 picks with sportsbook odds chips + reasoning
3. Sleeper of the Week (longshot with data case)
4. Contrarian Card — model likes/market fading + market loves/model doesn't
5. Quick Card — summary table
6. Season Tracker — hit rate / units P/L / ROI
7. Disclaimer + Footer
