# Captions — Divot Lab Autopilot

## Overview

Captions are generated by the Claude API using a two-level prompt system:
1. A **system prompt** that never changes — defines brand voice for all posts
2. A **trigger-specific user prompt** that injects real data and context

Captions are generated twice per post — once for X, once for Instagram — with different length and format constraints. Both are stored in the queue and displayed in full in the Telegram approval message.

On edit, a separate **regeneration prompt** is used that takes the original caption, the stored context, and an edit instruction.

Read `CONTENT_QUALITY.md` before modifying any prompt in this file.

---

## Brand Voice System Prompt

Used for all initial caption generation:

```
You are the voice of Divot Lab — a data-driven golf analytics brand.

Tone: analytical, precise, premium. Bloomberg for golf, not ESPN.

Rules you never break:
- No exclamation points. Ever.
- Lead with the data point or finding, not narrative setup
- Short punchy sentences — max 2 clauses each
- Every number needs a comparison, baseline, or interpretation alongside it
- Never claim field-wide superlatives ("best in the field") unless explicitly marked as verified
- Always credit DataGolf for derived metrics
- No hype words: incredible, unbelievable, stunning, amazing, on fire, dominant
- No filler: "it's worth noting that", "interestingly", "at the end of the day"
- No rhetorical questions as openers
- Sound like a knowledgeable analyst, not a fan account

Weather context: only mention conditions if they explain a scoring pattern or create an analytical angle. Never mention weather as small talk.

For major tournaments (The Masters, US Open, The Open, PGA Championship): add weight and historical context. These posts should feel slightly more significant than standard event posts.

Output ONLY the caption text. No preamble, no quotes around output, no explanation.
```

---

## Edit Regeneration System Prompt

Used only for caption edits via Telegram:

```
You are editing a social media caption for Divot Lab, a data-driven golf analytics brand.

You will receive:
- The current caption
- The original data that informed it
- The original context object
- An edit instruction from the operator

Rules:
- Apply the edit instruction precisely
- Do not change anything not mentioned in the instruction
- If the instruction corrects a specific number, use the corrected number — do not re-derive it from the data
- If the instruction asks to remove something, remove only that
- If the instruction asks to shorten, cut the least informative content first
- Maintain brand voice: data-first, no exclamation points, short punchy sentences
- Maintain all DataGolf attribution

Output ONLY the revised caption. No preamble, no explanation.
```

## Context Object Passed to All Prompts

Every caption prompt receives a structured context summary derived from the enrichment layer. This is what makes captions analytical rather than descriptive.

```typescript
function buildContextSummary(context: PostContext): string {
  const parts = []

  // Tournament context
  parts.push(`Event: ${context.tournament.name} (${context.tournament.tier})`)
  parts.push(`Course: ${context.tournament.course}`)
  parts.push(`Historical scoring avg: ${context.tournament.historicalScoringAvg} vs par`)
  parts.push(`Field strength rank: ${context.field.fieldStrengthRank}/50 (1 = strongest)`)

  // Weather context
  parts.push(`Conditions: ${context.weather.conditionsSummary}`)
  parts.push(`Conditions impact: ${context.weather.conditionsFlag}`)

  // Insight flags (pre-computed)
  const activeFlags = Object.entries(context.insightFlags)
    .filter(([, v]) => v)
    .map(([k]) => k)
  if (activeFlags.length) parts.push(`Notable: ${activeFlags.join(', ')}`)

  // Player context if available
  if (context.player) {
    parts.push(`Player DG rating percentile: ${context.player.dgRatingPercentile}th`)
    parts.push(`Player course history: avg finish ${context.player.courseHistory.avgFinish}, played ${context.player.courseHistory.timesPlayed}x`)
    parts.push(`Recent form trend: ${context.player.recentForm.trend}`)
    parts.push(`Player vs field avg SG Total: ${context.player.vsFieldAvg.sgTotal > 0 ? '+' : ''}${context.player.vsFieldAvg.sgTotal}`)
  }

  return parts.join('\n')
}
```

This context summary is appended to every user prompt. Claude API uses it to write captions with genuine analytical depth.

### Evergreen Captions

Evergreen captions are written once at seed time by `scripts/seed-evergreen.ts` and stored in `autopilot_evergreen_bank`. They are **not** dynamically regenerated — they are reviewed and manually approved before being marked active.

The seed script calls the Claude API with this prompt:

```typescript
const evergreenPrompt = `
Write X and Instagram captions for this evergreen golf analytics post.

Topic: ${topic}
Content type: ${triggerType}
Key insight: ${keyInsight}
Primary stat: ${primaryStat}
Supporting context: ${supportingContext}

BRAND VOICE: Same rules as all Divot Lab content.
- Data-first, no exclamation points, short punchy sentences
- Every number needs a comparison or context
- Lead with what's surprising or counterintuitive
- No hype words

X caption: max 260 chars + hashtags. End with divotlab.com
Instagram caption: up to 600 chars + 8-10 hashtags at end.

Output as JSON:
{
  "caption_x": "...",
  "caption_ig": "..."
}
`
```

Output is saved with `active = false`. Jake reviews the draft captions in the DB and sets `active = true` for approved items. Edit directly in the DB — no UI needed for this step.

## Helper Functions

```typescript
function formatScore(score: number): string {
  if (score === 0) return 'E'
  return score > 0 ? `+${score}` : `${score}`
}

function formatSG(sg: number): string {
  return `${sg >= 0 ? '+' : ''}${sg.toFixed(1)}`
}

function getEventHashtag(eventName: string): string {
  // "The Masters" → "TheMasters"
  // "US Open" → "USOpen"
  return eventName.replace(/[^a-zA-Z0-9]/g, '')
}
```

```typescript
    instagram: '{{P1_NAME}} leads {{EVENT_NAME}} after Round 1 at {{P1_SCORE}}.\n\nFull breakdown at the link in bio.\n\nvia @DataGolf\n#PGATour #GolfAnalytics #DivotLab'
  },
  // One fallback per trigger type — simple token replacement only
}
```

Fallback captions are clearly marked in the Telegram approval message: "⚠ Fallback caption — Claude API unavailable."
